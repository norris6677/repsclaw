## CLI 使用指南

### 启动测试工具
```bash
npm run test:hospital:cli
# 或
npx tsx tests/interactive/hospital-test-cli.ts
```

### 可用命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `list` | 列出所有测试用例 | `list` |
| `info <id>` | 查看测试用例详情 | `info TC-001` |
| `run <id>` | 运行指定测试用例 | `run TC-001` |
| `run-all` | 运行所有测试 | `run-all` |
| `run-cat <cat>` | 按类别运行 | `run-cat basic` |
| `report` | 生成测试报告 | `report` |
| `demo` | 运行交互式演示 | `demo` |
| `help` | 显示帮助 | `help` |
| `exit` | 退出 | `exit` |

### 类别代码
- `basic` - 基础功能测试
- `error` - 错误处理测试
- `scenario` - 场景测试
- `boundary` - 边界条件测试
- `performance` - 性能测试

## 测试报告

测试报告会自动生成在 `tmp-test-cli/test-report.json`，包含：
- 执行时间戳
- 测试汇总统计
- 每个测试用例的详细结果
- 总耗时和通过率

## 扩展测试用例

如需添加新的测试用例，在 `hospital-test-cli.ts` 的 `TEST_CASES` 数组中添加：

```typescript
{
  id: 'TC-XXX',
  name: '测试名称',
  description: '测试描述',
  category: 'basic',
  priority: 'high',
  steps: [
    { action: 'subscribe', params: { name: '测试医院' } },
    {
      action: 'list',
      params: {},
      validate: (result, service) => ({
        passed: result.data.hospitals.length === 1,
        message: '应订阅1家医院'
      })
    }
  ],
  expectedResult: '期望的结果描述'
}
```
