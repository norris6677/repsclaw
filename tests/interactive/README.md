# 医院订阅功能 - 交互式测试方案

本目录提供了**无需部署到 OpenClaw**即可测试医院订阅功能交互的多种方案。

## 快速开始

```bash
# 🆕 新版 CLI 测试工具（推荐 - 16+ 测试用例）
npm run test:hospital:cli

# 1. CLI 交互式测试（基础版）
npm run test:interactive:cli

# 2. 聊天模拟器（模拟真实聊天体验）
npm run test:interactive:chat

# 3. Web 界面（可视化操作）
npm run test:interactive:web

# 4. 自动化场景测试（CI/CD）
npm run test:interactive:scenario
```

## 方案 4: 自动化场景测试

### 启动
```bash
npx tsx tests/interactive/scenario-runner.ts
# 或
npm run test:interactive:scenario
```

### 预定义场景

#### 场景 1: 新用户首次使用完整流程
- 检查首次使用状态
- 订阅第一家医院（自动默认）
- 订阅第二家医院
- 查看列表
- 更改主要医院

#### 场景 2: 多医院管理和切换
- 订阅 4 家医院
- 设置医院B为主要
- 取消医院B（验证自动转移）
- 验证剩余 3 家

#### 场景 3: 错误处理和边界情况
- 空医院名称（应报错）
- 取消不存在的订阅（应报错）
- 设置未订阅医院为主要（应报错）
- 重复订阅（应成功更新）

#### 场景 4: 每日提示逻辑测试
- 检查未提示状态
- 更新提示日期
- 检查已提示状态

## 扩展新的交互测试

### 添加新的场景测试
编辑 `scenario-runner.ts`，在 `scenarios` 数组中添加：

```typescript
{
  name: '我的新场景',
  description: '场景描述',
  steps: [
    {
      action: 'subscribe',
      params: { name: '测试医院' },
      expected: {
        status: 'success',
        conditions: [
          (result) => result.data.hospitals.length === 1
        ]
      }
    },
    // ... 更多步骤
  ]
}
```

### 添加新的聊天意图
编辑 `chat-simulator.ts`，在 `parseIntent` 函数中添加：

```typescript
if (/搜索.*医院|查找.*医院/.test(lower)) {
  return { intent: 'search', data: { keyword: extractKeyword(input) } };
}
```

|-------|---------|--------------|-----------|
| 代码位置 | `tests/unit/` | `tests/interactive/` | `tests/interactive/` |
| 运行命令 | `npm run test:unit` | `npm run test:hospital:cli` | `npm run test:interactive:*` |
| 测试对象 | 函数/方法 | 完整功能场景 | 用户交互流程 |

| 测试报告 | ✅ | ✅ | ❌ |

| 运行环境 | 无头 | 可交互 | 可交互 |
| 目的 | 验证正确性 | **验证功能完整性** | 验证体验 |

### 何时使用新版 CLI 测试工具？

✅ **推荐使用场景：**
- 新功能开发完成后的功能验证
- 回归测试（运行全部用例）
- 按类别执行测试（如只测错误处理）
- 生成测试报告提交QA
- 调试单个失败的测试用例

❌ **不适合的场景：**
- 需要模拟真实用户体验（用聊天模拟器）
- 需要可视化界面演示（用 Web 界面）

## 建议的测试流程

1. **开发阶段**: 使用 `hospital-test-cli` 运行测试用例验证功能
2. **功能验证**: 使用 `cli-interactive` 快速手动验证
3. **联调阶段**: 使用 `chat-simulator` 验证交互逻辑
4. **演示阶段**: 使用 `http-server` 进行产品演示
5. **提测阶段**: 使用 `hospital-test-cli run-all` 进行完整回归测试
6. **CI/CD**: 集成 `hospital-test-cli report` 生成测试报告
